# Sentinel Journal 🛡️

## 2025-05-14 - XSS via innerHTML Entity Decoding **Vulnerability:** Using `document.createElement('textarea')` and setting its `innerHTML` to decode HTML entities is vulnerable to Cross-Site Scripting (XSS). If the input contains a closing `</textarea>` tag followed by malicious HTML (e.g., `<script>` or `<img>` with `onerror`), it will be executed when added to the DOM. **Learning:** This pattern is often used as a "quick hack" to decode HTML entities, but it treats the input as HTML instead of just text, allowing for tag injection if the input isn't strictly controlled. **Prevention:** Use `DOMParser` to parse the input as `text/html` and then retrieve the `textContent` from the resulting document. This safely decodes entities while ignoring or stripping any HTML tags without executing them.
